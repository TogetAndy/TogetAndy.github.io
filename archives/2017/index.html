1.1 The progress of abstraction

    • c /fortran /BASIC /primary abstraction still requires you to think in terms of the structure of the computer rather than the structure of the problem you are trying to solve.

    • Early languages such as LISP and APL chose particular views of the world (“All problems are ultimately lists” or “All problems are algorithmic”).

    • Each of these approaches is a good solution to the particular class of problem they’re designed to solve, but when you step outside of that domain they become awkward.

    • The object-oriented approach goes a step farther by providing tools for the programmer to represent elements in the problem space

  • The fisrt successful O-O language (the language c++ base on) Smalltalk’s characteristics :

    • Everything is an object.

    • A program is a bunch of objects telling each other what to do by sending messages.

    • Each object has its own memory made up of other objects

    • every object has a type

    • every object of particular type can receive the same mesages

Crash_world

静则思，思则变，变则通，通则达。路是自己选的。旧世界即将崩塌，技术带来的变革。在不久的将来，世界将会走到到哪里去。

机器人，自动化，人工智能慢慢渗透进入我们的生活。技术取代人类现有工作职位，这只是时间问题。静止时相对的，运动是绝对的。变的主题一直都在我们这个社会上演，不管是茹毛饮血的原始社会，还是毫无自由的奴隶社会，亦或等级严格的封建社会，再到现如今的资本主义社会。社会一直在变革，熵增原理，一直在上演。人类社会正在变得日益复杂。这个复杂，即是社会体制上的，也是社会实体结构上的。

于个人，现在更为关注的是社会实体结构上的改变，是具体技术所带来的人们生活上的改变。去年AlphaGo战胜世界围棋冠军李世石。人工智能，机器学习算法战胜了人类。围棋的算法复杂度上来说，是我们（姑且说一般人）一般人难以思维的。这样复杂的问题计算机都能够解决。那么在其他方面，还有什么机器人不能干呢？你的学习能力，你的计算能力，思维能力是否能够赶上计算机。

随之带来的问题就是，很多职位将被机器人代替。失业在所难免。社会将会出现一大片无用的人。而反过来看待这个问题。那些人没有工作，没有购买力，他们将如何生存下去呢？姑且相信人类的生存能力，但是适者生存的进化论势必会无情的上演。

技术动摇社会结构，将社会分为有技术的人和没有技术的人。（作为业界的人看待这个问题感觉会有片面之感）。随之而来的额便是贫富差距的拉大。
《华盛顿邮报》做过一个统计，1978年到2011年，美国工人的平均工资只增长了6%，CEO 的平均收入增长了727%。技术的变革一直在进行。工资差距被拉开，这种趋势只会越来越大。

社会流动性减弱。以前和你竞争的是人，现在和你竞争的是机器。你再努力也不可能超过机器，超不过软件。效率，成本你都无法和机器相竞争。造成的结果就是，通过奋斗走到社会上层的难度越来越大。

尤瓦尔·赫拉利将这种情景，列为21世纪最悲惨的威胁之一：

“随着人工智能变得越来越聪明，会有更多的人被挤出就业市场。没人知道大学该学什么，因为没人知道20岁的时候学的东西到了40岁还有没有用。等你知道的时候，已经有数十亿人变得一点用都没有了。这不是偶然，而是必然。”

世界上有没有安全的工作？

公务员可能比较安全，因为这个职业改变得很缓慢，而且没有技术升级的压力。医生和律师，也比较安全，因为对于这些行业，经验很重要，但技术正在把它们的成本降下来。厨师和物流，也是比较安全的行业，因为烧菜机器人和送货机器人，实现成本很高，人力实现比较便宜，暂时不会被取代。但是低端的厨师和物流是纯粹的体力劳动，非常辛苦，没有进入门槛，供给非常大，拿不到高工资。

最终来说，人类社会的就业形态正在发生深刻的改变，“终生职业”越来越少了。每个人都应该尽早打算，如果明天你的职业消失了，你该怎么办？

大学重某个角度来说失去了他的意义。你在大学里能学到真正的知识，也不应该在那里待四年。如果只学最需要学习的东西，一年就够了。读大学，只是18岁时很多种选择中的一种，不是唯一的选择，更谈不上是最好的选择。校园是一个美丽的地方，但是如果一定要在里面待上四年，那还是算了吧。（一年学完4年的知识，这似乎有点夸张，然而这样的做法还没有人去尝试。）

作为规划，人们应该更多的发展这些能力。个性化服务，创造创意能力，决策能力，人格魅力。这些机器人所代替不了的能力。

在理解互联网世界之前，我们需要了解它的一些重要协议。而这些协议工作在不同的层。
前辈们将互联网这样分层：

physical layer

link layer

network layer

transport layer

application layer

每一层都会完成相应的功能，我们都知道计算机网络最底层传输的是二进制0 1 信号。跟二进制联系相关度最高的就是physical layer 。我们的互联网需要将不同的pc连接起来。而物理性的连接，如通过光缆、电缆、双绞线、无线电。而这些容易让我们想到的什么呢，是的，网络的电气特性。传送 0 1 电信号。

单纯的脉冲信号是没有任何意义的，计算机之所以能够读懂一连串的的0 1 脉冲信号，在于我们人为的规定了特定的解析方式，

而linker layer 便在physical layer 基础上确定了0 1 的分组方式，

但是问题也来了。每个人的思维理解方式不同。不同的公司的工程师在思考如何解决这个问题的时候，最后用到的方法绝对不会一模一样。那他们两家公司产品出现交集，软件的统一性上便会出现问题。那么如何解决呢？

所以出现了一些统一的规定，这就是以太网协议（针对电信号分组）
一组电信号构成一个数据包，called frame 帧每一帧分两部分 标头head 和data

标头会对数据帧的一些数据进行描述比如发送者，接受者。data则是数据包的具体内容。

标头长度固定 18个字节。 data 46-1500字节

mac地址：主机的物理标识地址
mac地址是单一无二的，长度是48个二进制位。通常由12个16进制数表示。

经营模式的思考

近来，读到阮一峰先生的《未来世界的幸存者》一书。其中的一章让我想起了在汇通的经历。

  • 创业，简单两个字，好写却不好做。如果你想创业，首先你需要问一问你有没有这个能力。你有没有足够的启动资金。你对你将要从事的行业有多少了解。你的个人素质能否胜任创业者这一角色。这是起初我对于创业的一些见解。我认为，在没有想清楚这些问题之前。还是不要有创业的想法。

  • 以前我认为资金是最重要的，但我错了，找到你的经营模式是最重要的。

    有人也许会说，你必须先搞到一笔钱，没钱怎么能创业！话是这么说，但是这并不是你的头等大事，还有一件事比融资更重要。那就是在搞到钱之前，你必须先想好怎么使用这些钱。创业的第一步，就是你要想好你到底要干什么。在动身之前，你必须确定前进的方向。创业可不是旅行，你不能先把背包收拾好，然后再考虑去哪里。这就是为什么在融资之前，创业者都要先写一份《项目计划书》的原因。

    一般来说，在正式动手之前，你已经想好了自己要从事哪个行业。你的心里有一个基本的判断，你觉得由你来提供某种产品或者服务，应该是可以赚到钱的。那么，接下来的事情就很自然了。你就要筹划，如何把这种产品或服务生产出来，如何把它们送到消费者手中，如何才能取回货款。一句话，你要想好通过哪些步骤，你的“点子”可以变成现实。在管理学中，这些步骤就被称为“经营模式”。

  • “一句话，你要想好通过哪些步骤，你的“点子”可以变成现实。”

事实是点子确实变成了现实。但效果并不是所理想的。原因出在哪里呢。
一开始，我认为出在管理上。仓库分货出问题，错分问题，客服效率。
延误罚款，财务管理。

其实是输在第一步。在进入物流行业之前。并没与搞清楚到底要干什么，到底具体要干些什么的。只是简单的认为送快递就行了。没有具体的经营措施（也就是没有行业经验）。完全是靠着一股热情在干。

有热情是好的，但在面对一个月两，三万的亏损。需要的就不是热情了，而是一个能够处理问题的具体方案，而那个时候。我们根本没法解决。

很简单的一个问题，分拨中心那边每天错分十多个，一直没法解决。上面公司说这个东西，你们辛苦一下处理掉。恩，好。快递小哥不高兴了。这不归老子送，我不去。恩，好吧。退回分拨。第二天又下来了。恩，好的。客户不高兴了，投诉。恩，这下好了。责任追究到我们这，乖乖，罚款吧，亲。（一次500块哦）就是这样的问题。我们一直解决不了。

另一个问题，财务。开始不是朋友管理账目。每个月算出来，能够亏到3万。好吧，那就不玩了，拍拍屁股走人。过来要撤股。

还记得那一次，一个女的来店面拿快递。我当时在仓库，来的店面的时候。那个女的已经在那又哭又闹，说要投诉到总公司，投诉我们服务态度太差嘲笑消费者。我了解了一下情况。原来是因为来店面拿快递没人搭理她。后面又跟快递小哥发生争执。当时，朋友是想给自己人出头的。操蛋的是，最后我跟她好说歹说，赔礼道歉，出卖自己的底线（从来没有道歉半个小时的习惯好么）。

  • Action speak louder than words

    很多这样的事情。我们在看待一件事情的时候，通常来说都是关注到那些大的环节，很多细微的地方往往忽略。而这些具体而细微的地方往往是致命的。

当我们在看到文章的时候。我们往往是豪情万丈，经营模式我都懂了。创业不是很简单吗。是的，我知道我想干什么。但问题是，你对你想干的事情，是否有一个详细的调查规划。讲正式一点就是你有没有《项目计划书》。很多东西说起来想起来很简单。但做起来是另外一码事。
