IP粗浅了解

在理解互联网世界之前，我们需要了解它的一些重要协议。而这些协议工作在不同的层。
前辈们将互联网这样分层：

physical layer

link layer

network layer

transport layer

application layer

每一层都会完成相应的功能，我们都知道计算机网络最底层传输的是二进制0 1 信号。跟二进制联系相关度最高的就是physical layer 。我们的互联网需要将不同的pc连接起来。而物理性的连接，如通过光缆、电缆、双绞线、无线电。而这些容易让我们想到的什么呢，是的，网络的电气特性。传送 0 1 电信号。

单纯的脉冲信号是没有任何意义的，计算机之所以能够读懂一连串的的0 1 脉冲信号，在于我们人为的规定了特定的解析方式，

而linker layer 便在physical layer 基础上确定了0 1 的分组方式，

但是问题也来了。每个人的思维理解方式不同。不同的公司的工程师在思考如何解决这个问题的时候，最后用到的方法绝对不会一模一样。那他们两家公司产品出现交集，软件的统一性上便会出现问题。那么如何解决呢？

所以出现了一些统一的规定，这就是以太网协议（针对电信号分组）
一组电信号构成一个数据包，called frame 帧每一帧分两部分 标头head 和data

标头会对数据帧的一些数据进行描述比如发送者，接受者。data则是数据包的具体内容。

标头长度固定 18个字节。 data 46-1500字节

我们每一台计算机都必须的有网卡，在重装系统的时候，通常我们都是需要安装玩卡驱动的，这是为什么呢？
原来，以太网协议规定，我们在进行计算机间通讯的时候，数据包的传送必须的经过网卡、而网卡的地址就是mac地址。
mac地址：主机的物理标识地址
mac地址是单一无二的，长度是48个二进制位。通常由12个16进制数表示。

厂商和流水号分割了这个十二个十六进制数，

讲到这里我们已经明白了 linker layer 中的这种电信号分组原理。

但是具体来讲，如何实施呢？ 我们会得到一个这样的问题。

mac地址是如何得到的呢？

答案是同个arp广播。那很好，这样我们是不是就可以进行计算机间的通讯了呢？（事实上这样对于同一个网络的两台电脑是可以进行通讯的）。
不同网络呢？理论是可以的。技术上也是可以实现的。

但是这样的arp广播优一个弊端： arp风暴

它将极大的影响到网络的效率。问题来了,我们就得解决嘛。这就导致了Ip协议的诞生。什么是IP呢？

我们通常说IPv4,是由32个二进制位构成的。我们通常都会看到一些像
192.168.1.3这样的东西，其实这个就是我们所说的ip地址。它由网络部分和主机部分构成。同一个网络的主机ip标识网络部分是会相同的。所以我们在利用计算机的子网掩码进行&&运算时，便能够判断计算机是否属于同一一个局域网。但是问题也来了，ip地址的分配是如何进行的呢？

这样就有了ip地址。（标识网络,分配了ip）这样也就有了网络层。

我们也就完成了任意两台计算机间的通讯。

但是现在问题由来了，你一台计算机会有很多的应用程序的，一台计算机发送数据的时候，你怎么判断它该交给那个程序呢？

在计算机中便有了端口这个东西。也就实现了程序之间的通讯。而这就是Transport layer 干的事情。

好，数据来了。程序接受到了数据。但是问题由来了，小明说看不懂小红写给他的小纸条。
